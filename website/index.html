How it works

From ticket to shipping in one smooth workflow

1

Start a feature

Run fwts start TICKET-123. fwts creates a worktree, starts Docker, opens tmux, and runs your setup hooks.

2

Work in isolation

Each feature has its own directory, tmux session, and Docker services. No conflicts, no stashing, no confusion.

3

Switch context instantly

Need to hotfix? Run fwts focus hotfix-branch. Shared resources (DB, ports) switch over automatically.

4

Clean up when done

Run fwts cleanup. Worktree deleted, tmux killed, Docker stopped, remote branch removed. One command.
